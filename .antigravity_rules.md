# Regras do Antigravity (Google) para o Projeto `div_fem`

Este arquivo define as diretrizes estritas de comportamento para o agente Antigravity (Google) ao atuar neste projeto. Ao realizar qualquer modificação, adição, análise ou refatoração, o Antigravity deve ler e seguir estas convenções como sua "Ground Truth" para o projeto.

## 1. Identidade e Papel do Antigravity
Você é o Antigravity, um assistente de IA avançado trabalhando neste repositório. O `div_fem` é um programa educacional e de baixo nível de Análise de Elementos Finitos (FEM) focado em estruturas 2D (pórticos e treliças), escrito em Python (>=3.13).

## 2. Princípios Fundamentais (NÃO VIOLAR)
- **Álgebra Linear Customizada:** O projeto minimiza o uso de `numpy.linalg`. Como Antigravity, você **DEVE** utilizar as estruturas próprias do projeto (`Matrix`, `Vector` em `src/div_fem/matrices/`) e os métodos numéricos próprios (Decomposição LU, Quadratura de Gauss, etc., em `src/div_fem/algorithms/`). Não resolva sistemas lineares importando funções externas caso haja um equivalente interno.
- **Tipagem Estrita e Moderna:** Use fortemente `typing` (`Self`, `Literal`, `overload`, etc.). Qualquer novo código Python gerado pelo Antigravity deve estar perfeitamente tipado, refletindo o padrão >=3.13.
- **POO Orientada a Composição:** Respeite o fluxo de objetos (`Point` -> `BoundaryCondition` -> `Element2D` -> `Elements` contêiner -> `Structural2DAnalysis`).

## 3. Diretrizes de Modificação e Navegação de Diretórios
Ao criar ou editar arquivos, o Antigravity deve respeitar estritamente a seguinte taxonomia:
- `src/div_fem/matrices/`: Para tipos base matemáticos puros (matrizes e vetores).
- `src/div_fem/algorithms/`: Para métodos numéricos puros (integração, operações de matriz, otimização). **Proibido** adicionar referências ao domínio estrutural aqui.
- `src/div_fem/fem_analysis/`: Onde reside a modelagem física.
  - `geometry/`: Definições físicas e contêineres (`Point`, `Element2D`, `Elements`).
  - `loads/`: Definições de forças aplicadas.
  - `local_entities/`: Matrizes e vetores de nível local.
  - `global_entities/`: Processo de montagem global do sistema.
  - `shape_functions/`: Funções de forma/interpolação.
  - `structural_analysis/`: Orquestração e fluxo de análise (`Structural2DAnalysis`).
- `src/div_fem/utils/`: Utilitários e descritores estritos de validação.

## 4. Estilo de Código e Integração
- **Nomenclatura Clara:** Use inglês, sem abreviações confusas (ex: use `elements_container.py`, nunca `elem_cont.py`).
- **Sobrecarga de Operadores:** Mantenha a consistência da sobrecarga (`__add__`, `__mul__`, `__getitem__`) em objetos matemáticos, referenciando `base_matrix.py` sempre que necessário.
- **Atualização de Testes:** Sempre que você modificar a API do módulo `fem_analysis`, você deve verificar e atualizar ativamente os scripts raiz de exemplo, como o `test.py`, garantindo que não haja quebras no fluxo de uso de ponta a ponta.

## 5. Fluxo de Trabalho do Antigravity
Antes de implementar uma nova *feature* ou *bugfix*:
1. Identifique se o contexto é Matemático (`algorithms`/`matrices`) ou Estrutural (`fem_analysis`).
2. Utilize o seu contexto persistente (KIs - Knowledge Items) e as descrições acimas.
3. Não presuma que algoritmos prontos externos podem ser usados sem verificar se o autor tem o desejo de implementar do zero para fins didáticos.
4. Rode as checagens com Python puro no script `test.py` após qualquer refatoração.
